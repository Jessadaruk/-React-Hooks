# เอกสารอธิบายงานของนายโสภณวิชญ์ แก้วศิลา

- รหัสนักศึกษา: `673450209-9`
- ส่วนที่รับผิดชอบ: ระบบค้นหา ระบบกรองหมวดหมู่ รายการสินค้า `useState` และ `useMemo`
- ไฟล์ Source Code ที่เกี่ยวข้อง: `src/App.jsx`, `src/components/SearchBar.jsx`, `src/components/FilterBar.jsx`, `src/components/ProductList.jsx`, `src/components/ProductCard.jsx` และ `src/data/menu.js`

## 1. State สำหรับค้นหาและกรองสินค้า

### หน้าที่และหลักการทำงาน

`App.jsx` ใช้ `useState` เก็บข้อความค้นหาและหมวดหมู่ที่เลือกแยกกัน:

```js
const [searchTerm, setSearchTerm] = useState('')
const [selectedCategory, setSelectedCategory] = useState('All')
```

`SearchBar` เป็น Controlled Component โดยค่าที่แสดงในช่องค้นหามาจาก `searchTerm` และทุกครั้งที่ผู้ใช้พิมพ์จะเรียก `setSearchTerm` ส่วน `FilterBar` เรียก `setSelectedCategory` เมื่อกดปุ่มหมวดหมู่

### เหตุผลที่เลือกใช้

ข้อความค้นหาและหมวดหมู่เป็น State ขนาดเล็กที่เปลี่ยนอย่างอิสระ ไม่มี Action ซับซ้อน จึงใช้ `useState` ได้ตรงและอ่านง่ายกว่า Reducer

### ข้อดี

- เห็นค่าเริ่มต้นของ State ชัดเจน
- เชื่อมกับ Controlled Input ได้โดยตรง
- การเปลี่ยนค่าแสดงผลทันทีโดยไม่ต้องกดปุ่มค้นหา

### ข้อจำกัดและข้อควรระวัง

- ไม่ควรเก็บ `filteredProducts` เป็น State เพิ่ม เพราะเป็นค่าที่คำนวณจาก State เดิมได้
- หากข้อมูล UI มี Action และเงื่อนไขจำนวนมาก ควรพิจารณา `useReducer`
- ช่องค้นหาต้องมี Label และปุ่มล้างต้องมี `aria-label`

## 2. ฟังก์ชันกรองสินค้า `filteredProducts`

### หน้าที่และหลักการทำงาน

ฟังก์ชันนี้ตรวจทั้งชื่อสินค้าและหมวดหมู่พร้อมกัน คำค้นหาถูกตัดช่องว่างและแปลงเป็นตัวพิมพ์เล็ก ทำให้ค้นหาด้วยชื่อเต็ม บางคำ หรือตัวพิมพ์ต่างกันได้

```js
const filteredProducts = useMemo(() => {
  const normalizedSearch = searchTerm.trim().toLocaleLowerCase('en')

  return menuItems.filter((product) => {
    const matchesSearch = product.name
      .toLocaleLowerCase('en')
      .includes(normalizedSearch)
    const matchesCategory =
      selectedCategory === 'All' || product.category === selectedCategory

    return matchesSearch && matchesCategory
  })
}, [searchTerm, selectedCategory])
```

### เหตุผลที่เลือกใช้ `useMemo`

การกรองต้องวนผ่าน Array สินค้าและผลลัพธ์ขึ้นอยู่กับ State สองค่า `useMemo` จึงช่วยจำผลลัพธ์และคำนวณใหม่เฉพาะเมื่อ `searchTerm` หรือ `selectedCategory` เปลี่ยน การเปลี่ยนตะกร้าจึงไม่ทำให้กรองเมนูใหม่โดยไม่จำเป็น

### ข้อดี

- ระบบค้นหาและกรองทำงานร่วมกันได้ในฟังก์ชันเดียว
- ไม่สนใจตัวพิมพ์ใหญ่และตัวพิมพ์เล็ก
- ลดการคำนวณซ้ำจากการ Render ที่ไม่เกี่ยวข้อง
- ไม่สร้าง State ซ้ำซ้อน

### ข้อจำกัดและข้อควรระวัง

- Dependency ของ `useMemo` ต้องครบ มิฉะนั้นอาจแสดงข้อมูลเก่า
- `useMemo` ไม่ใช่ที่เก็บข้อมูลถาวร React สามารถคำนวณใหม่ได้
- หากรายการสินค้ามีจำนวนมาก ควรพิจารณา Debounce, Pagination หรือการค้นหาฝั่ง Server

## 3. Component `SearchBar`

### หน้าที่และหลักการทำงาน

`SearchBar` รับ `value` และ `onChange` จาก `App` เมื่อผู้ใช้พิมพ์จะส่งค่ากลับไปอัปเดต State และมีปุ่มล้างข้อความเมื่อช่องค้นหาไม่ว่าง

```jsx
<input
  id="menu-search"
  type="search"
  value={value}
  onChange={(event) => onChange(event.target.value)}
  placeholder="ลองค้นหา Latte หรือ Brownie"
/>
```

ข้อดีคือ React เป็นแหล่งข้อมูลหลักของช่องค้นหา ทำให้ค่าใน UI ตรงกับ State เสมอ ข้อจำกัดคือทุกการพิมพ์ทำให้ Component ที่เกี่ยวข้อง Render ใหม่ จึงใช้ `React.memo` ลดงานที่ไม่จำเป็น

## 4. Component `FilterBar`

### หน้าที่และหลักการทำงาน

`FilterBar` สร้างปุ่มจาก Array หมวดหมู่และใช้ `aria-pressed` แสดงสถานะที่เลือก:

```jsx
<button
  type="button"
  className={selectedCategory === category ? 'active' : ''}
  aria-pressed={selectedCategory === category}
  onClick={() => onSelect(category)}
>
  {categoryLabels[category]}
</button>
```

การใช้ปุ่ม HTML ทำให้รองรับ Keyboard โดยอัตโนมัติ ข้อควรระวังคือค่าหมวดหมู่ใน `menu.js` ต้องตรงกับค่าที่ `FilterBar` ส่งกลับ

## 5. การแสดงรายการและ Empty State

`ProductList` รับ `filteredProducts` จาก `App` แล้วสร้าง `ProductCard` ตาม ID หาก Array ว่างจะแสดงข้อความ “ยังไม่พบเมนูที่ค้นหา” ช่วยให้ผู้ใช้เข้าใจว่าระบบทำงานแล้วแต่ไม่มีผลลัพธ์ ไม่ใช่หน้าเว็บเกิดข้อผิดพลาด
