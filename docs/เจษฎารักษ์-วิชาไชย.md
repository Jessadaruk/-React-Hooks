## 1. ฟังก์ชัน `cartReducer`

### หน้าที่และหลักการทำงาน

`cartReducer` เป็นฟังก์ชันหลักที่รับ State ตะกร้าปัจจุบันและ Action แล้วคืน State ใหม่ โดยไม่แก้ไข Array หรือ Object เดิมโดยตรง รองรับ Action ดังนี้:

- `ADD_ITEM` เพิ่มสินค้าใหม่ หรือเพิ่ม `quantity` เมื่อพบ ID เดิม
- `REMOVE_ITEM` ลบสินค้าตาม ID
- `INCREASE_QUANTITY` เพิ่มจำนวน 1 ชิ้น
- `DECREASE_QUANTITY` ลดจำนวน และลบรายการเมื่อลดจาก 1
- `CLEAR_CART` คืนตะกร้าเป็น Array ว่าง
- `HYDRATE_CART` นำข้อมูลที่ตรวจสอบแล้วจาก Local Storage เข้าสู่ Reducer

### ตัวอย่างโค้ดจริง

```js
case CART_ACTIONS.ADD_ITEM: {
  const existingItem = state.find((item) => item.id === action.payload.id)

  if (existingItem) {
    return state.map((item) =>
      item.id === action.payload.id
        ? { ...item, quantity: item.quantity + 1 }
        : item,
    )
  }

  return [...state, { ...action.payload, quantity: 1 }]
}
```

การใช้ `find` ตรวจรายการเดิมก่อนช่วยป้องกันสินค้าที่มี ID เดียวกันปรากฏซ้ำ ส่วน `map` และ Spread Syntax ทำให้การอัปเดตเป็นแบบ Immutable

### เหตุผลที่เลือกใช้

ตะกร้ามีหลาย Action และทุก Action ต้องรักษากฎร่วม เช่น ID ต้องไม่ซ้ำและจำนวนต้องไม่ต่ำกว่า 1 การรวมกฎไว้ใน Reducer ทำให้ตรวจสอบและทดสอบได้จากจุดเดียว เหมาะกว่าแยก `useState` หลายตัวไว้ตาม Component

### ข้อดี

- State transition มีชื่อชัดเจนและติดตามง่าย
- กฎของตะกร้าไม่กระจายอยู่หลาย Component
- ทดสอบแยกจาก UI ได้ใน `cartReducer.test.js`
- เพิ่ม Action ใหม่ในอนาคตได้ง่าย

### ข้อจำกัดและข้อควรระวัง

- Action และ Payload ต้องมีรูปแบบตรงกับที่ Reducer กำหนด
- Reducer ต้องเป็น Pure Function ห้ามเขียน Local Storage หรือแสดง Alert ภายใน
- `HYDRATE_CART` ต้องรับเฉพาะข้อมูลที่ผ่านการตรวจสอบแล้ว

## 2. ฟังก์ชันจัดการตะกร้าใน `CartProvider`

### หน้าที่และหลักการทำงาน

`CartProvider` สร้างฟังก์ชันที่ Component ลูกเรียกใช้ แล้วแปลงคำสั่งเหล่านั้นเป็น Action สำหรับ Reducer ได้แก่ `addItem`, `removeItem`, `increaseQuantity`, `decreaseQuantity` และ `clearCart`

```js
const addItem = useCallback((product) => {
  dispatch({ type: CART_ACTIONS.ADD_ITEM, payload: product })
}, [])

const removeItem = useCallback((productId) => {
  dispatch({ type: CART_ACTIONS.REMOVE_ITEM, payload: productId })
}, [])
```

`useCallback` ทำให้ Reference ของฟังก์ชันคงที่ และทำงานร่วมกับ `React.memo` ใน `ProductCard` และ `CartItem` เพื่อลดการ Render ที่ไม่จำเป็น

## 3. `CartContext` และ Custom Hook `useCart`

### หน้าที่และหลักการทำงาน

`CartContext` แชร์ตะกร้า ยอดรวม และฟังก์ชันจัดการตะกร้าให้ `Header`, `ProductCard`, `CartItem` และ `OrderSummary` โดยไม่ต้องส่ง Props ผ่าน Component หลายชั้น

```js
export function useCart() {
  const context = useContext(CartContext)

  if (context === undefined) {
    throw new Error('useCart must be used within a CartProvider')
  }

  return context
}
```

### เหตุผลที่เลือกใช้

ข้อมูลตะกร้าถูกใช้งานในหลายส่วนของหน้าเว็บ การใช้ Context ลดปัญหา Props Drilling ส่วน `useCart` รวมวิธีเข้าถึง Context และตรวจสอบ Provider ไว้ในจุดเดียว

### ข้อดี

- Component เรียกใช้ API ของตะกร้าได้โดยตรง
- ลดจำนวน Props ที่ Component ระหว่างทางไม่จำเป็นต้องใช้
- แจ้ง Error ที่เข้าใจง่ายเมื่อวาง Component ผิดตำแหน่ง

### ข้อจำกัดและข้อควรระวัง

- `useCart` ใช้ได้เฉพาะภายใน `CartProvider`
- Consumer อาจ Render เมื่อ Context Value เปลี่ยน จึงใช้ `useMemo`, `useCallback` และ `React.memo` ช่วยควบคุม
- ไม่ควรนำ State ทุกชนิดของหน้าเว็บมาเก็บใน Context เดียว

## 4. การคำนวณสรุปตะกร้า

`CartProvider` ใช้ `useMemo` คำนวณจำนวนสินค้าและยอดรวมจากตะกร้า:

```js
const totalItems = useMemo(
  () => cart.reduce((total, item) => total + item.quantity, 0),
  [cart],
)

const totalPrice = useMemo(
  () => cart.reduce(
    (total, item) => total + item.price * item.quantity,
    0,
  ),
  [cart],
)
```

ผลลัพธ์ถูกนำไปแสดงใน `Header` และ `OrderSummary` และจะคำนวณใหม่เฉพาะเมื่อข้อมูลตะกร้าเปลี่ยน
